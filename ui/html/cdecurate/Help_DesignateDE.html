Designating Data Elements
Designations Screen Overview

Before you go to the trouble of creating a new Data Element in your Context, you may choose to Designate an existing Data Element instead. Designating simply means that you will ‘point’ the existing Data Element (including all of its attributes) to your Context. In other words, the designated Data Element will now be ‘Used By’ your Context. A complete listing of Designation rules can be found on the NCI Center for Bioinformatics Business Rules page.

NOTE: Only Data Elements may be designated; and only those having the following workflow statuses: APPRVD FOR TRIAL USE, CMTE APPROVED, CMTE SUBMTD USED, DRAFT MOD, RELEASED, OR RELEASED-NON-CMPLNT.

NOTE: If you Designate a Data Element to your Context, you will have write privileges for only a few attributes for that Data Element in its current version. All other changes to the Data Element will be the responsibility of the "owning" context. This is usually the context for which the Data Element was originally created.

To designate a Data Element(s) using the CDE Curation Tool first, use the Search feature to locate the Data Element(s) that you would like to designate. Select the desired Data Element(s) from the search results by clicking on the box next to the Data Element's name. Then click the ‘Designations’ action button at the top of the main search screen. This will open the Designations screen. When you open the Designations screen all the currently designated items will appear in the appropriate fields. Creating Alternate Name Attributes

To Create an Alternate Name Attribute for your Context for the selected Data Element, first select an Alternate Name Type from the drop down list. Then Create an Alternate Name by entering text in the field provided. Click the Add Selection button. Your newly created Alternate Name Attributes will be displayed in the Selected Alternate Name and Type fields.

To Remove an Alternate Name Attribute from your context for the selected Data Element(s), highlight the desired Alternate Name and click the Remove Item button. If the selected Alternate Name is not in your context an error message will be displayed. Please note that Alternate Names must be removed individually.

Creating Reference Document Attributes

To Create Reference Document Attributes for your Context for the selected Data Element, first select a Reference Document Type from the drop down list. Then enter text for the Reference Document Name and Reference Preferred Question Text in the fields provided. If you wish to include a URL, simply type the desired URL into the field provided. Or you can click the Browse link to navigate to the desired URL and use the copy/paste method to select the URL and paste it into the field. Future releases will incorporate the Upload Document function. Once you have entered all the desired information, click the Add Selection button. Figures 7-8 show how to create Reference Document Attributes.

To Remove Reference Document Attributes from your context for the selected Data Element(s), highlight the desired Reference Document and click the Remove Item button. If the selected Reference Document is not in your context a message will be displayed. Please note that Reference Documents must be removed individually.

Classification Schemes and Classification Scheme Items

A Classification Scheme can be selected from the first drop down list provided in the "Classification Scheme and Classification Scheme Items" section. Please note that only Classification Schemes/Items from Contexts in which you have write permissions can be selected. The associated Classification Scheme Items (CSIs) will be displayed in the box to the right of the Classification Schemes. Children of the higherl level CSIs will be indented in a heirarchial manner in the Associated Classification Scheme Items box when appropriate. When desired CSIs are clicked on they and their Clssification Schemes are displayed in the boxes below. When the child is selected from the Class Scheme Items list, all its parents, up to top level, will be added in the selected list of Classification Scheme Items. Figures 9-11 show how to select Classification Schemes and Classification Scheme Items for Designated Data Elements.

How to add a Classification Scheme Item to a Data Element in your Context
  1. Select a Classification Scheme from the drop down ( ).
  2. The Classification Scheme Items are displayed in a hierarchical manner (when appropriate).
  3. Click the Update Used By Attributes button to to save the changes to the used by attributes ( ).
Note: Please make sure that the selected classification scheme's context is within the user's context

How to Remove a Classification Scheme Item from a Data Element in your Context
  1. Select the Classification Scheme or Classification Scheme Item.
  2. Click the Remove Item button to remove the selected Classification Scheme and Classification Scheme Item ( ).
  3. Click the Update Used By Attributes button to to save the changes to the used by attributes ( ).
Note: When a Classification Scheme Item is selected from the list of items for the selected Classification Scheme, it (and all its parents) are added to the Associated Classification Scheme Items.
Note: You may remove a higher level parent CSI without removing the children.

To remove Classification Schemes and associated CSIs, highlight the desired values one at a time and click the Remove Item button. You may remove higher level "parent" CSIs in this way without removing their children.
