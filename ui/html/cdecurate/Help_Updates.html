1. CDE Curation Tool V2.1 Enhanced Features
The CDE Curation Tool has expanded features and improved functionality in the Version 2.1 release. The sections below describe these features in detail.
1.1. Searching from Main Menu
   1.1.1.     Searching For
The CDE Curation Tool has extended its Search For capability to include the following:
  • Object Class
  • Property
   1.1.2.     Searching In
The CDE Curation Tool has extended its Search In capability to include the following:
  • Reference Document Text (applies to Data Elements only)
  • Reference Document Types (when Reference Document Text is selected for Data Elements)
  • Historical CDE ID (applies to Data Elements only)
  • Origin (applies to Data Elements, Data Element Concepts, Value Domains, and Conceptual Domains)
  • Permissible Value (applies to Data Elements and Value Domains)
  • Protocol ID/CRF name (wildcard functionality) for Data Elements
   1.1.3.    Simple and Advanced Search Filters
Simple and Advanced search filters are now available. These provide users with the ability to quickly and easily find desired Administered Components.

Simple Filter Components:
  • Context
  • Context Use (Owned By, Used By, Owned/Used By)
  • Version (radio buttons)
  • Workflow Status
  • Registration Status (applies to Data Element only)
  • Classification Schemes (applies to Classification Scheme Item only)
  • Value Domain Type (applies to Value Domain only)
  • Conceptual Domain (applies to Permissible Value only)
Advanced Filter Components:
  • Date Created
  • Creator
  • Date Modified
  • Modifier
  • Reference Document Types (when Reference Document Text is searched in Data Elements)
   1.1.4.    Search Result Functionality
  • Select/Unselect: All Search Results may be selected or unselected by clicking on the checkmark at the top of the checkbox column on the Search Results screen
  • Long Name is displayed as default for all Administered Components
  • Sorting: Blank records are sorted to the bottom of search results. For example, if you sort search results by Long Name, those records with blank Long Name fields will be displayed at the bottom of the search results.
   1.1.5.    Viewing Multiple Names
  • Users have the ability to view multiple Reference Document Text Types for Data Elements.
  • Users have the ability to view multiple Historical CDE ID for Data Elements.
  • Users may also view multiple Permissible Values associated to Data Elements and Value Domains.

   1.1.6.    Designating
Users may designate a single or block of Data Elements only in the 2.1 release.
1.2. NCI Enterprise Vocabulary Service (EVS) Search Functionality
  1.2.1.    EVS Search
The CDE Curation Tool Has expanded its EVS Search functionality to include the following:
  • Filter by EVS Source: This restricts the search results to a specific EVS source. (e.g. LOINC)
  • Search in Synonym
  • Search by Code vs. String
  • Ability to Set Metathesaurus Return Limit from 100-1000 Records
  • Ability to Search EVS when creating a new Value Meaning
  • Display Preferred Name for all Concepts searched from EVS
1.3. UI Design on Create/Edit/Validation Screens
The 2.1 CDE Curation Tool has a new look on all Create, Edit and Validation screens.
  1.3.1.    Mandatory vs. Optional Attributes

  • Mandatory attributes are represented with a red asterisk (*) on the Create and Edit screens
  • Mandatory and Optional sections have been removed from Create and Edit screens
  • Optional Attribute shading has been removed from Validation screens
1.4. Create/Edit Administered Components
   1.4.1.    Creating a Comment/Change Note
The user has the ability to enter a Comment or Change Note when Creating, Editing or Block Editing Administered Components.
   1.4.2.    Creating Derived Data Elements (for Data Element only)
A Derived Data Element can be created on the bottom section of the Create Data Element screen. A derived data element is the combination of two or more data elements, a representation type and a derivation rule.
   1.4.3.    Selecting a Registration Status (Data Element only)
A Registration Status can be selected from a drop down list when Creating or Editing a single Data Element. Registration Status is also an attribute to filter by on the Search screen for a Data Element.
   1.4.4.    Viewing Alternate Names (Data Element, Data Element Concept, Value Domain)
The user has the ability to view Alternate Names of Data Elements, Data Element Concepts and Value Domains on the Edit Existing and Version screens. Figures 1.4.4a-b illustrate this process.
