Using the Curation Tool to Create Sentinel Tool Alert Definitions

Overview of the Curation Tool Monitor/Unmonitor Feature

The caDSR Sentinel Tool, allows users to create and manage Alert Definitions for the caDSR. Alert Definitions are a set of rules that are periodically evaluated against the caDSR. If the conditions in those rules are met, notification is sent to the user (via email) in the form of a hyperlink to an activity report that specifies the changes that have taken place.

The Curation Tool allows you to create an Alert Definition for specific Data Elements, Data Element Concepts or Value Domains.

Monitoring an Administered Component

To monitor a Data Element, Data Element Concept or Value Domain search for the desired Administered Component (AC) and click the button from the search screen. This associates the AC with a Class Scheme Item setup for your user id. The Sentinel Tool then creates an Alert Definition to monitor that Class Scheme Item.

Unmonitoring an Administered Component

To remove the monitor for a Data Element, Data Element Concept or Value Domain select the desired AC and select the button from the search screen. This removes the association from the AC and the Class Scheme Item setup for your user id.

i The Alert Definition is not removed from the Sentinel Tool when the AC is unmonitored. The Alert will still monitor the Scheme Item setup for your user id. No ACs will be associated with it and no email will be sent.

Searching for Monitored Administered Components

In some cases you may wish to stop monitoring all ACs. In this case search for the Class Scheme Item "Monitor for *" and select the search result for your caDSR user account. Click the button to find Data Elements, Data Element Concepts and Value Domains associated with the monitored Class Scheme Item.
