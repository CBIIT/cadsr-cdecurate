Data Element Concepts (DEC) Overview

A Data Element Concept is the thing (or concept) in the real world that you want to represent. ISO/IEC 11179 defines Data Element Concept as an idea that can be represented in the form of a data element, described independently of any particular representation.

Like other primary functions of the Curation Tool, Data Element Concept curation features are accessed via the dark blue Navigation Bar at the top of the screen. Figure 1 illustrates where to access these features. You may also want to create a new Data Element Concept using an existing Data Element Concept as a starting template or version an existing Data Element Concept. Both of these options can be accessed via the Create èData Element Concept menu in the Navigation Bar.

Name a Data Element Concept

The ISO 11179 Standard describes a Data Element Concept as the combination of an Object Class, a Property, and Qualifiers for each. Definitions for each of these terms are documented below.

Data Element Concept
The ISO/IEC 11179 Standard describes a Data Element Concept as relating several data elements that record data about that concept with different representations. For example, a concept of PersonPlaceOfBirth is a Data Element Concept. A Data Element Concept is composed of an Object Class, a Property and optional Qualifiers for both.

Object Class
The ISO 11179 Standard describes an Object Class as ‘a set of ideas, abstractions, or things in the real world that can be identified with explicit boundaries and meaning and whose properties and behavior follow the same rules’. In other words, Object Classes are the ‘things about which you want to store and collect data.’ Examples of Object Classes include cars, persons, households, employees, etc. For the example of PersonPlaceOfBirth, an Object class is Person.

Property
The ISO 11179 Standard describes a Property as ‘a peculiarity common to all members of an object class’. In other words, properties are ‘what humans use to describe or distinguish Objects’. For the example given above, a Property is PlaceOfBirth.

Qualifiers
The ISO 11179 Standard describes qualifiers as being used to ‘differentiate between names (i.e. Object Class and Property combinations) that would otherwise be the same’. For the example of PersonPlaceOfBirth, a Qualifier of year is used to indicate a specific year.

Data Element Concepts are named by combining Primary and Qualifier Concepts for Object Classes and Properties. The Object Classes and Properties are then used for naming the Data Element Concept. This approach is new for version 3.0 of the CDE Curation Tool.

Building Blocks for Object Classes and Properties

Object Classes and Properties are the building blocks from which a Data Element Concept name is made. The CDE Curation Tool enables you to search caDSR and the NCI EVS (Enterprise Vocabulary Services) for Concepts. The Primary Concepts may represent existing Object Classes and Properties, or Concepts may be combined to create new Object Classes and Properties. Object Class and Property Long Names are also created from this interface.

An Object Class or Property may consist of one Primary Concept and one or more Qualifier Concepts. Each of these Concepts must be selected separately. Once selected the Concept's source database and Concept Code will be shown below the Concept. You may highlight Qualifier Concepts one at a time to see their source database and Concept Code.

The Object Class and Property Long Names will be constructed from the names of the Qualifier and Primary Concepts that have been chosen. The Long Name will be built left to right starting with the Concept at the top of the Qualifier Concepts list and ending with the Primary Concept. Once you have built the Object Class and Property Long Names they are not editable.

Please note that if you select only a Primary Concept from the caDSR database and select no Qualifier Concepts you have in effect selected an existing Object Class (or Property). In this case, no new Object Class (or Property) will be created in caDSR when you submit your Data Element Concept.

There are three options for creating a Data Element Concept Short Name. It is strongly suggested that you accept the default System Generated Short Name. Search for a Conceptual Domain

Conceptual Domain The ISO 11179 Standard describes Conceptual Domain as ‘a set of all possible, valid value meanings of a data element concept expressed without representation’. Conceptual Domains are plural, object oriented, lacking in representation, and may be associated with more than one Data Element Concept. For the example given above with the Data Element Concept of PatientPlaceOfBirth, a Conceptual Domain is Country.

When the Search link is activated, all Conceptual Domains in the selected Context are displayed. The user has the option to change the Context on the Search screen. Select Classification Scheme/Classification Scheme Items

Classification Schemes are a way of grouping Data Element Concepts based on common characteristics. Classifications also help add information about Data Element Concepts that is not easily included in definitions. For example, you might want to classify Data Element Concepts as belonging to the Classification Scheme Type of Disease. A Classification Scheme Item is a component of content in a Classification Scheme (i.e. Liver Cancer).

The parent-child relationships of Classification Scheme Items are displayed in a hierarchical order and also by indentation. Child Classification Scheme Items are indented farther than their parent. This is a new feature for version 3.0 of the CDE Curation Tool.

i Note that when you select a Classification Scheme Item that has parents then all of the parent and grandparent Classification Scheme Items will also be selected. Once selected, Classification Schemes and Classification Scheme Items can be removed by highlighting the desired item and then clicking the Remove Item button.
